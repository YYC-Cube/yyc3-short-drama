### 3. 用户注册登录流程

#### 3.1 用户注册流程

**流程图**
```
开始 → 选择注册方式 → 手机号注册/邮箱注册/第三方登录
  ↓
输入注册信息 → 验证码验证 → 密码强度验证
  ↓
检查用户是否存在 → 创建用户记录 → 生成JWT Token
  ↓
返回用户信息和Token → 注册成功 → 结束
```

**详细步骤**

1. **选择注册方式**
   - 用户选择注册方式：手机号、邮箱、第三方登录（微信、QQ、微博）
   - 系统显示对应的注册表单

2. **输入注册信息**
   - 手机号注册：输入手机号、验证码、密码、昵称
   - 邮箱注册：输入邮箱、验证码、密码、昵称
   - 第三方登录：跳转到第三方授权页面

3. **验证码验证**
   - 系统发送验证码到手机或邮箱
   - 用户输入验证码
   - 系统验证验证码有效性

4. **密码强度验证**
   - 系统验证密码强度（至少8位，包含大小写字母、数字、特殊字符）
   - 如果密码强度不足，提示用户修改

5. **检查用户是否存在**
   - 系统检查手机号或邮箱是否已注册
   - 如果已注册，提示用户直接登录

6. **创建用户记录**
   - 系统创建用户记录
   - 密码使用bcrypt加密存储
   - 设置用户状态为已激活

7. **生成JWT Token**
   - 系统生成JWT Token和Refresh Token
   - Token有效期设置为2小时
   - Refresh Token有效期设置为7天

8. **返回用户信息和Token**
   - 系统返回用户信息和Token
   - 客户端存储Token到本地

9. **注册成功**
   - 系统显示注册成功提示
   - 引导用户完善个人信息

**异常处理**

- 验证码错误：提示用户重新输入，最多尝试3次
- 密码强度不足：提示用户修改密码
- 用户已存在：提示用户直接登录
- 系统错误：显示错误提示，记录错误日志

#### 3.2 用户登录流程

**流程图**
```
开始 → 选择登录方式 → 手机号/邮箱登录/第三方登录
  ↓
输入登录信息 → 验证用户信息 → 检查用户状态
  ↓
验证密码 → 生成JWT Token → 更新最后登录时间
  ↓
返回用户信息和Token → 登录成功 → 结束
```

**详细步骤**

1. **选择登录方式**
   - 用户选择登录方式：手机号/邮箱、第三方登录
   - 系统显示对应的登录表单

2. **输入登录信息**
   - 手机号/邮箱登录：输入手机号或邮箱、密码
   - 第三方登录：跳转到第三方授权页面

3. **验证用户信息**
   - 系统查询用户信息
   - 检查用户是否存在

4. **检查用户状态**
   - 系统检查用户状态（已激活、已冻结、已删除）
   - 如果用户状态异常，提示用户联系客服

5. **验证密码**
   - 系统验证密码
   - 如果密码错误，提示用户重新输入，最多尝试5次

6. **生成JWT Token**
   - 系统生成JWT Token和Refresh Token
   - Token有效期设置为2小时
   - Refresh Token有效期设置为7天

7. **更新最后登录时间**
   - 系统更新用户最后登录时间
   - 记录登录IP和设备信息

8. **返回用户信息和Token**
   - 系统返回用户信息和Token
   - 客户端存储Token到本地

9. **登录成功**
   - 系统显示登录成功提示
   - 跳转到首页

**异常处理**

- 用户不存在：提示用户注册
- 密码错误：提示用户重新输入，最多尝试5次
- 用户状态异常：提示用户联系客服
- 系统错误：显示错误提示，记录错误日志

### 4. 内容浏览流程

#### 4.1 首页浏览流程

**流程图**
```
开始 → 加载首页 → 获取推荐内容
  ↓
显示推荐内容 → 用户滑动浏览 → 加载更多内容
  ↓
用户点击短剧 → 跳转到短剧详情页 → 结束
```

**详细步骤**

1. **加载首页**
   - 系统加载首页数据
   - 显示加载动画

2. **获取推荐内容**
   - 系统调用推荐接口获取个性化推荐内容
   - 推荐算法基于用户画像、热门内容、相似内容

3. **显示推荐内容**
   - 系统显示推荐短剧列表
   - 每个短剧显示封面、标题、简介、标签

4. **用户滑动浏览**
   - 用户滑动浏览推荐内容
   - 系统记录用户浏览行为

5. **加载更多内容**
   - 用户滑动到底部时，系统加载更多内容
   - 显示加载动画

6. **用户点击短剧**
   - 用户点击感兴趣的短剧
   - 系统跳转到短剧详情页

**异常处理**

- 网络错误：显示网络错误提示，提供重试按钮
- 加载失败：显示加载失败提示，提供重试按钮
- 无推荐内容：显示空状态，引导用户探索其他内容

   - 显示加载动画

### 5. 内容创作流程

#### 5.1 创建短剧流程

**流程图**
```
开始 → 点击创建短剧 → 填写短剧信息
  ↓
上传封面海报 → 选择分类标签 → 提交审核
  ↓
AI预审核 → 人工审核 → 审核结果通知
  ↓
审核通过 → 发布短剧 → 结束
```

**详细步骤**

1. **点击创建短剧**
   - 用户点击创建短剧按钮
   - 系统跳转到创建短剧页面

2. **填写短剧信息**
   - 用户填写短剧标题、简介、总集数、单集时长
   - 系统验证信息完整性

3. **上传封面海报**
   - 用户上传短剧封面和海报
   - 系统验证图片格式和大小

4. **选择分类标签**
   - 用户选择短剧分类和标签
   - 系统提供分类和标签推荐

5. **提交审核**
   - 用户点击提交审核按钮
   - 系统创建短剧记录（草稿状态）

6. **AI预审核**
   - 系统进行AI预审核（敏感词、违规内容检测）
   - 如果发现违规内容，标记为待人工审核

7. **人工审核**
   - 管理员进行人工审核
   - 审核通过后，短剧状态更新为已通过

8. **审核结果通知**
   - 系统通知创作者审核结果
   - 如果审核未通过，提供修改建议

9. **审核通过**
   - 创作者点击发布按钮
   - 系统发布短剧

10. **发布短剧**
    - 系统更新短剧状态为已发布
    - 推送短剧到推荐系统

**异常处理**

- 信息不完整：提示用户完善信息
- 图片格式错误：提示用户上传正确格式的图片
- AI审核未通过：提示用户修改内容
- 人工审核未通过：提示用户根据审核意见修改内容
- 系统错误：显示错误提示，记录错误日志

#### 5.2 AI剧本生成流程

**流程图**
```
开始 → 点击AI剧本生成 → 选择生成参数
  ↓
输入主题角色 → 构建Prompt → 调用AI模型
  ↓
解析生成结果 → 格式化剧本 → 保存剧本草稿
  ↓
预览剧本 → 用户确认 → 保存剧本 → 结束
```

**详细步骤**

1. **点击AI剧本生成**
   - 用户点击AI剧本生成按钮
   - 系统跳转到AI剧本生成页面

2. **选择生成参数**
   - 用户选择生成参数：主题、类型、风格、集数、单集时长
   - 系统提供参数推荐

3. **输入主题角色**
   - 用户输入剧本主题、主要角色、情节大纲
   - 系统验证输入信息

4. **构建Prompt**
   - 系统根据用户输入构建Prompt模板
   - Prompt包含主题、角色、风格、长度等信息

5. **调用AI模型**
   - 系统调用AI模型（GPT-4/Claude-3）
   - 等待AI生成结果

6. **解析生成结果**
   - 系统解析AI生成的剧本内容
   - 提取剧集、场景、台词等信息

7. **格式化剧本**
   - 系统将剧本格式化为标准结构
   - 生成剧本预览

8. **保存剧本草稿**
   - 系统保存剧本草稿
   - 用户可以继续编辑

9. **预览剧本**
   - 用户预览生成的剧本
   - 可以进行修改和调整

10. **用户确认**
    - 用户确认剧本内容
    - 点击保存按钮

11. **保存剧本**
    - 系统保存剧本
    - 剧本状态更新为已完成

**异常处理**

- 输入信息不完整：提示用户完善信息
- AI生成失败：显示生成失败提示，提供重试按钮
- 解析失败：显示解析失败提示，提供重试按钮
- 系统错误：显示错误提示，记录错误日志

### 6. 社交互动流程

#### 6.1 发表评论流程

**流程图**
```
开始 → 点击评论按钮 → 输入评论内容
  ↓
敏感词检测 → 提交评论 → 保存评论记录
  ↓
更新评论数 → 通知相关用户 → 评论成功 → 结束
```

**详细步骤**

1. **点击评论按钮**
   - 用户点击评论按钮
   - 系统显示评论输入框

2. **输入评论内容**
   - 用户输入评论内容
   - 系统验证评论长度（1-500字）

3. **敏感词检测**
   - 系统进行敏感词检测
   - 如果发现敏感词，提示用户修改

4. **提交评论**
   - 用户点击提交按钮
   - 系统提交评论

5. **保存评论记录**
   - 系统保存评论记录
   - 评论状态设置为正常

6. **更新评论数**
   - 系统更新短剧或剧集的评论数
   - 实时更新评论数显示

7. **通知相关用户**
   - 系统通知短剧创作者有新评论
   - 如果是回复评论，通知被回复用户

8. **评论成功**
   - 系统显示评论成功提示
   - 评论显示在评论列表中

**异常处理**

- 评论内容为空：提示用户输入评论内容
- 评论内容过长：提示用户缩短评论内容
- 发现敏感词：提示用户修改评论内容
- 系统错误：显示错误提示，记录错误日志

**流程图**
```
开始 → 点击点赞按钮 → 检查点赞状态
  ↓
未点赞 → 添加点赞记录 → 更新点赞数
  ↓
已点赞 → 取消点赞记录 → 更新点赞数
  ↓
通知创作者 → 点赞成功 → 结束
```

**详细步骤**

1. **点击点赞按钮**
   - 用户点击点赞按钮
   - 系统检查点赞状态

2. **检查点赞状态**
   - 系统查询用户是否已点赞
   - 如果未点赞，执行点赞操作
   - 如果已点赞，执行取消点赞操作

3. **未点赞**
   - 系统添加点赞记录
   - 更新点赞状态为已点赞

4. **已点赞**
   - 系统删除点赞记录
   - 更新点赞状态为未点赞

5. **更新点赞数**
   - 系统更新短剧或剧集的点赞数
   - 实时更新点赞数显示

6. **通知创作者**
   - 如果是点赞操作，通知创作者
   - 如果是取消点赞，不通知创作者

7. **点赞成功**
   - 系统更新点赞按钮状态
   - 显示点赞成功提示

**异常处理**

- 系统错误：显示错误提示，记录错误日志
- 网络错误：显示网络错误提示，提供重试按钮

### 7. 支付订阅流程

#### 7.1 付费解锁流程

**流程图**
```
开始 → 点击付费解锁 → 选择解锁内容
  ↓
计算解锁费用 → 创建订单 → 调用支付接口
  ↓
用户完成支付 → 支付成功回调 → 更新订单状态
  ↓
解锁内容 → 通知用户 → 解锁成功 → 结束
```

**详细步骤**

1. **点击付费解锁**
   - 用户点击付费解锁按钮
   - 系统显示付费解锁页面

2. **选择解锁内容**
   - 用户选择要解锁的剧集或短剧
   - 系统显示解锁费用

3. **计算解锁费用**
   - 系统计算解锁费用
   - 显示费用明细

4. **创建订单**
   - 系统创建订单记录
   - 订单状态设置为待支付

5. **调用支付接口**
   - 系统调用支付接口（微信支付、支付宝）
   - 生成支付二维码或跳转到支付页面

6. **用户完成支付**
   - 用户完成支付
   - 支付平台返回支付结果

7. **支付成功回调**
   - 系统接收支付成功回调
   - 验证支付结果

8. **更新订单状态**
   - 系统更新订单状态为已支付
   - 记录支付时间和交易ID

9. **解锁内容**
   - 系统解锁用户访问权限
   - 更新用户解锁记录

10. **通知用户**
    - 系统通知用户解锁成功
    - 发送解锁成功消息

11. **解锁成功**
    - 用户可以观看解锁的内容
    - 显示解锁成功提示

  ↓
更新星值等级 → 返回星值变化 → 星值获取成功 → 结束

**详细步骤**

1. **用户完成行为**
   - 用户完成特定行为（观看、分享、创作等）

   - 系统验证行为有效性
   - 防止刷星值行为

   - 系统根据行为类型计算星值奖励
   - 不同行为对应不同星值奖励

4. **更新用户星值余额**
   - 系统更新用户星值余额
   - 增加相应星值

   - 记录获取原因和时间

   - 如果达到新等级，显示升级提示

   - 显示星值获取提示

8. **星值获取成功**
   - 用户星值增加
   - 显示星值获取成功提示

- 行为无效：不发放星值奖励
- 刷星值行为：封禁用户星值获取权限
